**Web Application Firewall (WAF)** protege suas aplicações contra ameaças como *SQL Injections*, *Remote File Inclusion (RFI)*, *Cross-Site Scripting (XSS)*, *Cross-Site Request Forgery (CSRF)*, *violações de protocolo* (como body vazio em POST), *Code Injection* e outras vulnerabilidades da web, incluindo as [ameaças cibernéticas mais comuns](https://www.azion.com/pt-br/learning/websec/como-waf-protege-contra-ameacas-ciberneticas/#quais-sao-as-ameacas-ciberneticas-mais-comuns). O WAF analisa requisições HTTP e HTTPS, detecta e bloqueia atividades maliciosas antes que elas cheguem à infraestrutura da sua aplicação.

O WAF é um módulo do [Firewall](/pt-br/documentacao/secure/firewall/) que opera na sétima camada do modelo OSI, a camada de aplicação, funcionando como uma barreira para filtrar e monitorar o tráfego entre sua aplicação e as requisições da internet. Ele se baseia em uma metodologia de *pontuação* (score) por requisição: cada requisição HTTP/HTTPS é comparada a um conjunto rigoroso de padrões de aplicações e recebe um *score* associado a uma família de ameaças. De acordo com o score, a requisição pode ser liberada ou bloqueada diretamente nos edge nodes da Azion, antes que a ameaça atinja sua origem ou cause qualquer tipo de dano.

O WAF pode operar em dois modos, **Learning** ou **Blocking**, definidos em uma regra do [Rules Engine](/pt-br/documentacao/secure/firewall/reference/rules-engine/) com o behavior **Set WAF Rule Set** ou via [API da Azion](https://api.azion.com/#ae9cdca8-3949-4b25-b1e6-2e071cd193fc). Leia os [conceitos do WAF](/pt-br/documentacao/secure/waf/concepts/) para entender o modelo de score, os modos de operação e como o WAF interpreta as requisições.

Você pode gerenciar seus WAF Rule Sets pelo [Azion Console](https://console.azion.com) ou pela [API da Azion](https://api.azion.com/#ae9cdca8-3949-4b25-b1e6-2e071cd193fc).

## Principais recursos

### WAF Main Settings

A seção **Main Settings** de um WAF Rule Set é configurada ativando a proteção contra famílias específicas de ameaças — SQL Injection, Remote File Inclusions (RFI), Directory Traversal, Cross-Site Scripting (XSS), File upload, Evading tricks, Unwanted Access e Identified Attack — e definindo um nível de sensibilidade: Lowest, Low, Medium, High e Highest. É possível ativar e desativar a proteção para cada família de ameaças individualmente através do switch **Active**.

Veja as tabelas de referência de [Tipos de ameaças](/pt-br/documentacao/secure/waf/reference/rule-sets/#tipos-de-ameacas) e [Níveis de sensibilidade](/pt-br/documentacao/secure/waf/reference/rule-sets/#niveis-de-sensibilidade).

### WAF Tuning

**WAF Tuning** é uma ferramenta analítica que exibe as requisições detectadas para cada regra do WAF, agrupadas pelo **rule ID**, com informações adicionais como **total de hits**, **número de URIs**, **IPs únicos** e mais, para auxiliar na configuração das **Allowed Rules** do WAF e garantir a melhor calibração para sua aplicação. Você pode filtrar por **Domínio**, **Data**, **Network Lists**, **IP** e **Países**.

Veja [Como fazer o tuning do WAF](/pt-br/documentacao/secure/firewall/guides/tune-waf/).

### Allowed Rules e Exceções

Se tráfego interno, testes e falsos positivos estiverem sendo bloqueados pelo WAF, você pode criar *Allowed Rules* (referenciadas como **Exceções** na API) para permitir explicitamente padrões legítimos de requisição. Cada regra do WAF possui um identificador numérico único, chamado `Rule ID`, que você pode usar para identificar, filtrar e permitir/bloquear comportamentos específicos da sua aplicação.

Veja a [referência de Exceções do WAF](/pt-br/documentacao/secure/waf/reference/custom-allowed-rules/) para os campos, as match zones, a lista completa de regras internas com seus `Rule IDs` e o gerenciamento via API.

## Limites e planos

Para configurar um **WAF Rule Set**, que é como uma configuração de WAF é chamada, você precisa ter uma configuração de [Firewall](/pt-br/documentacao/secure/firewall/guides/firewall-definir-main-settings/) com o módulo **Web Application Firewall** ativado.

:::tip
**Aumente limites** <br></br>
Você pode solicitar o aumento dos limites com base no seu plano. Contate o [time de suporte técnico](/pt-br/documentacao/services/support/) para fazer a solicitação.
:::

Estes são os **limites default**:

| Escopo | Limite |
| ------ | ------ |
| Tamanho de logs de Debug Rules | 1.5 kB |
| Consultas de WAF Tuning | 3 dias |
| Tamanho máximo do payload da requisição | 160 KB |

:::note
**Limite de Tamanho do Payload da Requisição**  
Payloads maiores que 160 KB não serão analisadas pelo WAF e acionarão a regra 2 do WAF.
:::
