**Real-Time Events** allows you to query and analyze raw logs in real time. Each data source exhibits detailed information for logs from events related to access, behavior, and performance of products connected to your applications.

Once your applications begin receiving traffic, you'll be able to diagnose data through Real-Time Events.

<LinkButton link="/en/documentation/observe/real-time-events/" label="go to real-time events reference" severity="secondary" target="_blank" />

To start generating data and diagnose your logs:

- [Build an application](/en/documentation/get-started/journeys/launch/): create an application and, optionally, configure domains and modules, in addition to functions.
- [Secure an application with WAF](/en/documentation/get-started/journeys/protect/): use Firewall and the Web Application Firewall (WAF) module to protect your applications against threats.
- [Configure a stream](/en/documentation/observe/data-stream/guides/use-data-stream/): create and configure a stream and [choose variables](/en/documentation/observe/data-stream/guides/data-stream-select-variables/) to stream your data to other platforms.
- [Collect navigation data](/en/documentation/observe/data-stream/guides/analyze-navigation-data/): use Edge Pulse to collect navigation data through your browser pages.
- [Understand logs](/en/documentation/observe/data-stream/guides/understand-logs/): after creating your application and configuring other products, understand how to analyze the logs exhibited on Real-Time Events.
- [Add filters](/en/documentation/observe/data-stream/guides/add-filters-events/): set filters to query more accurate results in your log inspection on Real-Time Events.
