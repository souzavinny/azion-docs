**Firewall** protects workloads running on Azion's globally distributed infrastructure by inspecting and filtering traffic close to the requester—before it reaches the application. You use it to block unwanted requests, mitigate DoS and DDoS attacks, protect against OWASP Top 10 threats, filter traffic by IP/CIDR, ASN, or geolocation, apply rate limits, and run custom security logic at the edge.

Its composable architecture lets you combine modules—**DDoS Protection**, **Web Application Firewall**, **Network Shield**, **Bot Manager**, and **Functions**—to address specific aspects of traffic inspection and threat mitigation:

- [DDoS Protection](/en/documentation/secure/ddos-protection/): mitigates Distributed Denial of Service attacks at the network and application levels. Activated by default, with no additional configuration.
- [Web Application Firewall](/en/documentation/secure/waf/): protects applications against threats such as SQL Injection, Remote File Inclusion (RFI), Cross-Site Scripting (XSS), and other OWASP Top 10 threats.
- [Network Shield](/en/documentation/secure/network-shield/): creates filters by IP/CIDR, ASN, or countries (geolocation) through Network Lists.
- [Bot Manager](/en/documentation/secure/bot-manager/): implements bot mitigation techniques, including blocklists, fingerprints, tampering protection, brute force prevention, advanced rate limiting, and human challenge.
- [Functions](/en/documentation/secure/firewall/reference/functions/): runs serverless security functions—Azion curated or your own code—for extended functionalities such as IP reputation, JWT, and credential stuffing protection.

A firewall configuration consists of an identification name, the domains it protects, the activated modules, and the security rules configured in the **Rules Engine** tab. Each rule combines *Criteria* (conditionals) and *Behaviors* (commands), and the modules you activate define which criteria and behaviors are available. Rules run sequentially until the request is blocked or restricted, or until all rules are processed—the request only reaches your application if no rule blocks or rejects it. You can share the same firewall among all applications that adopt the same security policy.

You can manage your firewalls through [Azion Console](https://console.azion.com) or the [Azion API](https://api.azion.com).

:::note
You can clone an existing firewall through the Azion API. The cloned firewall has identical settings as the original one, including main settings, functions instances, and rules from Rules Engine—but the domains associated with the original firewall aren't copied. Retrieve the [ID of the firewall](https://api.azion.com/#090fa5b0-3d68-4521-9a90-f4d93773f6d9) you want to clone and run a [POST request to clone it](https://api.azion.com/#7e92d69a-c98c-4578-b920-38a13e1a3bd2), providing a new name.
:::

| Scope | Limit |
| ----- | ----- |

| ASN in a network list | 20,000 lines |
